# BAINSA Dashboard — Codebase Reference

Read this file instead of exploring the codebase. It is complete and up-to-date as of 2026-03-26.

## What this is

Instagram Story approval dashboard for BAINSA. Displays stories generated by the "Sofia" AI agent as 9:16 card previews. Users can edit, approve/reject stories, browse by date, and view agent output in a slide-out drawer.

Live at `dashboard.pavelj.com` (Cloudflare tunnel -> localhost:3000). Systemd service: `bainsa-dashboard`.

## Tech stack

- **Next.js 16** (App Router) + TypeScript + React 19
- **Tailwind CSS 4** with `@theme inline` design tokens in `globals.css`
- **No database** — filesystem only (markdown files + JSON sidecars)
- **Auth**: single shared password, httpOnly cookie, `proxy.ts` guard
- **Font**: Space Grotesk (placeholder for Alliance No.1/No.2)
- **Zero external dependencies** beyond Next.js + Tailwind

## Important gotchas

- **SECURE_COOKIES=true** -> auth cookie has `Secure` flag -> `macmini:3000` (HTTP) won't work. Always use `dashboard.pavelj.com` (HTTPS).
- **Sandy Bridge CPU** -> no Turbopack, production build only (`npm run build`), `npm run dev` is painfully slow.
- **Server Actions** must redirect to final URLs (no double-redirect).
- **Rebuild required** after `.env.local` changes.
- **`export const dynamic = 'force-dynamic'`** needed on pages using `new Date()` to prevent static caching at build time.

## Environment variables (`.env.local`)

| Var | Purpose |
|---|---|
| `DASHBOARD_PASSWORD` | Login password (plaintext, compared in Server Action) |
| `DASHBOARD_TOKEN` | Pre-computed HMAC stored in cookie for verification |
| `SECURE_COOKIES` | `"true"` -> Secure flag on auth cookie |
| `STORIES_PATH` | Sofia's story markdown files dir |
| `MARCO_HANDOFFS_PATH` | Marco's handoff markdown files dir |
| `APPROVALS_PATH` | Approval JSON sidecar files dir (same as STORIES_PATH) |
| `OPENCLAW_PASSWORD` | OpenClaw gateway password for Sofia chat API |

## Data flow

1. **Sofia** (OpenClaw agent) writes `YYYY-MM-DD.md` to `STORIES_PATH`
2. **Marco** (OpenClaw agent) writes `YYYY-MM-DD.md` to `MARCO_HANDOFFS_PATH`
3. Dashboard reads these files, parses stories, renders as cards
4. Edits write back to the same `.md` file via `replaceStory()`
5. Approvals stored in `YYYY-MM-DD.approved.json` sidecar next to stories

## Story markdown format (Sofia output)

```markdown
---
## Story 1: Title Here

**Division:** Analysis
**Accent color:** #fe6203
**Layout:** headline-top-body-source

**Headline:** The punchy headline text

**Body:** Body text up to ~240 chars

**Source tag:** SourceName

**Corner accent:** >
```

Sections separated by `\n---\n`. Parser also supports embedded JSON blocks (future format).

## Data types (`types/index.ts`)

```typescript
interface Story {
  index: number;           // 1-based
  title: string;           // from "## Story N: Title"
  division: "Analysis" | "Projects" | "Culture" | string;
  accentColor: string;     // hex
  layoutTemplate: string;
  headline: string;
  body: string;
  sourceTag: string;
  cornerAccent: ">" | "+";
}

interface ApprovalState {
  approved: number[];   // 1-based story indices
  rejected: number[];
}

interface ComplianceResult {
  colorValid: boolean;    // accentColor matches division canonical
  headlineOk: boolean;    // <= 80 chars
  bodyOk: boolean;        // <= 240 chars
  sourcePresent: boolean;
  pass: boolean;
}

const ACCENT_COLORS = { Analysis: "#fe6203", Projects: "#fe43a7", Culture: "#fe43a7" };
```

## Responsive behavior

- **Desktop (>=1024px)**: 4-column grid, card scale computed from column width (capped at 1.0)
- **Tablet (<1024px)**: auto-fill columns, min 280px each
- **Mobile (<640px)**: cards at 0.8 scale, header uses CSS grid (2 rows), editor stacks vertically (preview hidden), drawer full-width

## Auth flow

1. User visits any page -> `proxy.ts` checks `bainsa-auth` cookie
2. No valid cookie -> redirect to `/login`
3. Submit password -> Server Action compares against `DASHBOARD_PASSWORD` env var
4. Match -> sets `bainsa-auth` cookie (HMAC token, 7-day expiry) -> redirect to today's stories
5. Mismatch -> redirect to `/login?error=1`
6. Rate limit: 5 failed attempts -> 15min lockout (in-memory)

## API security

All mutation endpoints (`update`, `approve`) require `X-Requested-With: fetch` header to prevent CSRF from form submissions.

## Sofia inline chat

The story editor has an integrated chat panel where you can ask Sofia to refine stories via natural language. Architecture:

- **Session persistence**: OpenClaw sessions keyed by `dashboard:edit:{date}:{index}:{uuid}`. UUID stored in localStorage (`sofia-session:{date}:{index}`). Reset button generates new UUID.
- **API flow**: Client -> `POST /api/stories/[date]/[index]/chat` -> OpenClaw gateway (`localhost:18789/v1/chat/completions`) with `model: openclaw/story-generator` and `x-openclaw-session-key` header.
- **Response parsing**: Sofia responds with explanation + optional JSON code block. `parseResponse()` extracts the JSON and applies field updates to the draft.
- **Layout**: Desktop = preview | form | chat (3 columns). Mobile = FIELDS/SOFIA tab switcher.

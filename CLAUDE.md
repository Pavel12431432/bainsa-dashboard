# BAINSA Dashboard ‚Äî Codebase Reference

Read this file instead of exploring the codebase. It is complete and up-to-date as of 2026-03-28.

## What this is

Instagram Story approval dashboard for BAINSA. Displays stories generated by the "Sofia" AI agent as 9:16 card previews. Users can edit stories (manually or via inline Sofia chat), approve/reject, browse by date, search across all dates, view version history, and chat with Marco/Sofia agents in slide-out drawers.

Live at `dashboard.pavelj.com` (Cloudflare tunnel -> localhost:3000). Systemd service: `bainsa-dashboard`.

## Tech stack

- **Next.js 16** (App Router) + TypeScript + React 19
- **Tailwind CSS 4** with `@theme inline` design tokens in `globals.css`
- **No database** ‚Äî filesystem only (markdown files + JSON sidecars)
- **Auth**: single shared password, httpOnly cookie, `proxy.ts` guard
- **Font**: Space Grotesk (placeholder for Alliance No.1/No.2)
- **Zero external dependencies** beyond Next.js + Tailwind

## Important gotchas

- **SECURE_COOKIES=true** -> auth cookie has `Secure` flag -> `macmini:3000` (HTTP) won't work. Always use `dashboard.pavelj.com` (HTTPS).
- **Sandy Bridge CPU** -> no Turbopack, production build only (`npm run build`), `npm run dev` is painfully slow.
- **Server Actions** must redirect to final URLs (no double-redirect).
- **Rebuild required** after `.env.local` changes.
- **`export const dynamic = 'force-dynamic'`** needed on pages using `new Date()` to prevent static caching at build time.
- **No Co-Authored-By** in commit messages ‚Äî user explicitly doesn't want this.
- **OpenClaw HTTP API doesn't work** on version 2026.3.23 ‚Äî use `docker exec openclaw openclaw agent` CLI instead. See `lib/openclaw.ts`.
- **Card scaling**: StoryCard uses `useLayoutEffect` + `ResizeObserver` to scale cards. Do NOT try CSS-only approaches ‚Äî `scale(calc(cqw))` produces a length not a number, and `tan(atan2(cqw, px))` breaks on mobile browsers. The current JS approach runs before paint (no CLS).
- **Sofia format**: SOUL.md must use concrete examples, not template hints like `(max 8 words)` ‚Äî Sofia copies hints literally into output, breaking the parser.
- **Mobile keyboard + fixed panels**: `h-dvh`, `interactive-widget`, and `visualViewport` API all failed to fix iOS keyboard pushing slide panels. Left as a known issue for now.

## Environment variables (`.env.local`)

| Var | Purpose |
|---|---|
| `DASHBOARD_PASSWORD` | Login password (plaintext, compared in Server Action) |
| `DASHBOARD_TOKEN` | Pre-computed HMAC stored in cookie for verification |
| `SECURE_COOKIES` | `"true"` -> Secure flag on auth cookie |
| `STORIES_PATH` | Sofia's story markdown files dir |
| `MARCO_HANDOFFS_PATH` | Marco's handoff markdown files dir |
| `APPROVALS_PATH` | Approval JSON sidecar files dir (same as STORIES_PATH) |
| `OPENCLAW_PASSWORD` | OpenClaw gateway password for Sofia chat API |

## Data flow

1. **Sofia** (OpenClaw agent) writes `YYYY-MM-DD.md` to `STORIES_PATH`
2. **Marco** (OpenClaw agent) writes `YYYY-MM-DD.md` to `MARCO_HANDOFFS_PATH`
3. Dashboard reads these files, parses stories, renders as cards
4. Edits write back to the same `.md` file via `replaceStory()`
5. Approvals stored in `YYYY-MM-DD.approved.json` sidecar next to stories
6. Version history stored in `YYYY-MM-DD.history.json` sidecar next to stories

## Story markdown format (Sofia output)

```markdown
---
## Story 1: Title Here

**Division:** Analysis
**Accent color:** #fe6203
**Layout:** headline-top-body-source

**Headline:** The punchy headline text

**Body:** Body text up to ~240 chars

**Source tag:** SourceName

**Corner accent:** >
```

Sections separated by `\n---\n`. Parser also supports embedded JSON blocks (future format).

## Data types (`types/index.ts`)

```typescript
interface Story {
  index: number;           // 1-based
  title: string;           // from "## Story N: Title"
  division: "Analysis" | "Projects" | "Culture" | string;
  accentColor: string;     // hex
  layoutTemplate: string;
  headline: string;
  body: string;
  sourceTag: string;
  cornerAccent: ">" | "+";
}

interface ApprovalState {
  approved: number[];   // 1-based story indices
  rejected: number[];
}

interface ComplianceResult {
  colorValid: boolean;    // accentColor matches division canonical
  headlineOk: boolean;    // > 0 and <= 60 chars
  bodyOk: boolean;        // > 0 and <= 180 chars
  sourcePresent: boolean;
  pass: boolean;
}

interface HistoryEntry {
  story: Story;
  label: string;
  timestamp: string;
}

const ACCENT_COLORS = { Analysis: "#fe6203", Projects: "#fe43a7", Culture: "#fe43a7" };
```

## Design tokens

Defined in `globals.css` as CSS vars + Tailwind `@theme inline`:

- **Colors**: `brand-black` (#0a0a0a), `brand-white` (#f4f3f3), `accent-analysis` (#fe6203), `accent-projects` (#fe43a7), `accent-culture` (#fe43a7), `surface` (#111111), `border` (#1a1a1a), `border-light` (#222222), `border-mid` (#333333), `muted` (#666666), `success` (#22c55e), `danger` (#ef4444)
- **Card**: 405x720px (9:16), 24px border-radius, 28px padding, 3px accent top border
- **Typography**: headline 2rem/600, body 1rem/400/70% opacity, source 0.75rem/600, logo 1.125rem/600

## Responsive behavior

- **Desktop (>=1024px)**: 4-column grid, cards auto-scale to fill column width (capped at 405px)
- **Tablet (<1024px)**: auto-fill columns, min 280px each
- **Mobile (<640px)**: single column centered (max 405px), MARCO/SOFIA buttons hidden (accessible via hamburger), editor stacks vertically with FIELDS/SOFIA tab switcher

## Auth flow

1. User visits any page -> `proxy.ts` checks `bainsa-auth` cookie
2. No valid cookie -> redirect to `/login`
3. Submit password -> Server Action compares against `DASHBOARD_PASSWORD` env var
4. Match -> sets `bainsa-auth` cookie (HMAC token, 7-day expiry) -> redirect to today's stories
5. Mismatch -> redirect to `/login?error=1`
6. Rate limit: 5 failed attempts -> 15min lockout (in-memory)

## API security

All mutation endpoints require `X-Requested-With: fetch` header (CSRF prevention). Date params validated with `isValidDate()` regex (path traversal prevention).

## Sofia inline chat (story editor)

- **Session persistence**: OpenClaw sessions keyed by `dashboard-{date}-{index}-{uuid}`. UUID in localStorage. Reset generates new UUID.
- **API flow**: Client -> `POST /api/stories/[date]/[index]/chat` -> `chatWithAgent("sofia", ...)` via Docker exec.
- **Response parsing**: `parseResponse()` extracts JSON field updates from Sofia's markdown.
- **Accept/Revert**: Suggestions show inline diff (strikethrough old ‚Üí green new). User explicitly accepts or reverts.

## Agent chat (drawer)

- **Entry points**: Header buttons (collapsed output, chat mode) or hamburger menu Agent Outputs (expanded output, read mode).
- **Session**: Per agent+date UUID in localStorage. Messages persisted in localStorage.
- **API**: `POST /api/agent-chat` -> `chatWithAgent(agent, sessionId, message)`. Returns raw text reply.
- **Output block**: Collapsible with draggable resize handle between output and chat area.
- **After reply**: Refetches `/api/agent-status` so output block updates if agent modified files.

## Version history

- **Storage**: `YYYY-MM-DD.history.json` sidecars, keyed by story index, capped at 50 entries.
- **Labels**: "Original", "Sofia: headline, body", "Manual edit", "Restored: previous version".
- **Original snapshot**: Uses immutable `originalStory` ref set on mount.

## Story search

- **API**: `GET /api/search?q=term` scans all `YYYY-MM-DD.md` files, searches headline/body/title/sourceTag (case-insensitive). Max 20 results, newest first.
- **UI**: Search bar in hamburger menu, debounced 300ms, min 2 chars. Results show headline + date as links.
- **Highlight**: Navigates to `/stories/{date}?highlight={index}`. StoryGrid scrolls to card and pulses orange outline (2s fade).

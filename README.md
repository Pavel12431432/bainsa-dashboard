# BAINSA Dashboard

Instagram Story approval dashboard for BAINSA (Bocconi AI & Neuroscience Student Association). Displays stories generated by the "Sofia" AI agent as 9:16 card previews. Edit, approve/reject, browse by date, search, version history, PNG export, agent chat, A/B variants, and a Teach Sofia page for tuning her instructions.

Built with Next.js 16 (App Router) + React 19 + TypeScript + Tailwind 4. No database — all data is on disk as markdown + JSON sidecars.

## Quick start (demo mode, no OpenClaw needed)

Requires Node 20+.

```bash
git clone https://github.com/Pavel12431432/bainsa-dashboard.git
cd bainsa-dashboard
cp .env.example .env.local
npm install
npm run build && npm start
```

Open <http://localhost:3000>, log in with password `bainsa`.

The defaults in `.env.example` point at `samples/` (3 days of committed story data) and set `DEMO_MODE=true`, so the Marco/Sofia agents are stubbed with canned responses. Every feature works end-to-end — editing, variants, regenerate, chat — without Docker or a real AI backend.

### What demo mode stubs

- **Inline Sofia chat** (edit a story, talk to her) — returns a small canned headline tweak.
- **A/B variants** (EXPLORE tab) — returns 3 hardcoded variants (text / bullets / quote).
- **Regenerate** (stale banner) — bumps the stories file's mtime so the banner clears.
- **Agent drawer chat** (Marco / Sofia buttons in the header) — echoes back a demo message.

Force the stale banner to see Regenerate do something visible:

```bash
touch samples/handoffs/2026-04-20.md
```

Reload the page, click Regenerate, banner disappears.

## Project structure

```
app/                Next.js App Router pages + API routes
components/         React components (StoryGrid, StoryEditor, AgentChat, ...)
lib/                Parsing, serializing, auth, agent client, trackers
types/              Shared TypeScript types
samples/            Sample data for local dev (demo mode reads from here)
  stories/          Sofia's story markdown + variants sidecars
  handoffs/         Marco's daily research handoffs
  instructions/     Sofia's FIXED.md + ADAPTIVE.md
public/fonts/       Alliance No.1 / No.2 font files
```

See [CLAUDE.md](./CLAUDE.md) for a full codebase reference (data formats, file map, gotchas).

## Environment variables

Copy `.env.example` to `.env.local` and edit as needed. Full list documented in `.env.example`.

| Var | Purpose |
|---|---|
| `DASHBOARD_PASSWORD` | Login password |
| `DASHBOARD_TOKEN` | Any random string — stored in the auth cookie |
| `STORIES_PATH` | Where Sofia's story files live |
| `MARCO_HANDOFFS_PATH` | Where Marco's handoffs live |
| `APPROVALS_PATH` | Where approval sidecars are written (usually same as `STORIES_PATH`) |
| `SOFIA_INSTRUCTIONS_PATH` | Where `FIXED.md` + `ADAPTIVE.md` live |
| `DEMO_MODE` | `true` to stub all agent calls |
| `SECURE_COOKIES` | `true` only when serving over HTTPS |

## Production deploy

Production runs on a Mac mini via a Cloudflare tunnel at <https://dashboard.pavelj.com>, managed by a `bainsa-dashboard` systemd service. The dashboard shells out to an OpenClaw Docker container for real Marco/Sofia calls.

```bash
npm run build
sudo systemctl restart bainsa-dashboard
```

Teammates don't need this — demo mode covers the full UI.
